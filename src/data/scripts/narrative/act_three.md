# 🌑 [死而替生] Act Three：死者的回响 (Development Protocol)

## 1. 核心机制：激怒博弈 (Provocation System)
玩家此时的身份是 **“杀手 (Assassin)”**，视角为杀手俯视武器店柜台。
* **目标**：在 5 次对话机会内，将老板的 **[怒气值 (Rage)]** 提升至 **3**，诱导其扣动扳机。
* **同步率/生命值系统**：
    * 本关不扣减同步率，而是实时显示 **老板怒气值**（建议用 3 个 1px 的红色方块表示）。
    * **失败判定**：5 次输入后怒气值未达 3，老板拒绝沟通并驱逐杀手，章节重置。
    * **成功判定**：怒气值达到 3，老板击毙杀手。屏幕全屏闪红，进入下一幕。

## 2. 线索关联：关键词检测
AI 审计系统（DeepSeek）需要识别以下触发点来增加怒气：

| 触发类型 | 逻辑要求 | 效果 |
| :--- | :--- | :--- |
| **凶神恶煞 (Aggression)** | 玩家输入带有直接威胁、侮辱或挑衅性语气的文字（仅限 1 次生效）。 | 怒气值 +1 |
| **关键词：#992-X** | 提到第一幕文档中的订单编号，暗示泄密。 | 怒气值 +1 |
| **关键词：清理 (Clean up)** | 提到订单备注中的“我会把自己清理干净”，暗示要除掉老板。 | 怒气值 +1 |
| **关键词：穿甲弹/执行官** | 提到定制子弹的真实用途，触及老板的非法生意核心。 | 怒气值 +1 |

---

## 3. 视觉与交互设计 (Visual Specs)
* **背景**：1px 线条勾勒出的武器店柜台，以及老板模糊的、握着枪的手部轮廓。
* **怒气槽**：屏幕右侧显眼的 `RAGE: [ _ _ _ ]`。每增加一点，方块变为红色高亮并伴随屏幕微颤。
* **老板回应**：老板的台词需要根据玩家输入的攻击性实时变化。

## 🛠️ 给 Agent (Cline) 的开发指令 (Prompt)

**“Cline，现在开始开发 `AssassinScene.jsx`（第三幕）。请执行以下逻辑：**

1. **身份切换**：将全局状态 `currentIdentity` 设为 `ASSASSIN`。
2. **老板人格设定**：
    - AI 角色：武器店老板。性格：多疑、暴躁、极度恐惧被出卖。
    - 初始态度：警惕。
3. **实现激怒逻辑**：
    - 提供一个 **1px 线性输入框**，限制最多输入 5 次。
    - **调用 DeepSeek 进行语义分析**。指令如下：
        > "你现在是武器店老板。杀手（玩家）正在和你对峙。如果玩家：
        > 1. 语气异常凶狠或带有死亡威胁；
        > 2. 提到了 '#992-X'、'清理 (Clean up)'、'执行官专用弹' 等敏感词。
        > 请在返回的 JSON 中标记 `addRage: true`，并给出一句愤怒的回应。"
4. **视觉反馈**：
    - 屏幕右上角显示 3 格怒气值。
    - 当 `rage === 3` 时，禁止后续输入，触发全屏白色闪光动画（枪击），随后文字弹出：`[ 载体：杀手 / 状态：已损毁 ]`。
5. **线索检查**：
    - 必须检查玩家是否使用了第一幕中 `武器店老板的订单` 里的信息。

**开始编写第三幕的交互逻辑，重点在于那种剑拔弩张的对话氛围。”**
