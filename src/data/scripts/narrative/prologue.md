# 🌑 [死而替生] 序幕完整脚本 (Master Script)

## 场景：[ 序幕：理想的余晖 ]

### 1. 核心视觉架构
* **美学控制**：全黑背景 (`#000000`)，1px 白色线条 (`stroke-white/30`)。
* **交互方式**：**点击驱动**。玩家点击对话框或屏幕任何位置，触发下一段视觉/文案。
* **叙事核心**：从破碎的人性残留，到诡异的机械契约，最终完成非自愿的身份夺取。
* **全局视觉滤镜**：所有图片显示时必须覆盖 **1px 扫描线滤镜** 和 **微弱噪点**，确保二次元/插画风格与赛博朋克极简风格不冲突。

---

### 2. 剧情序列 (Phase-by-Phase)

#### **阶段一：冷漠的信道 (The Mobile Screen)**
* **视觉**：中央浮现 1px 线条勾勒的极简手机框。
* **交互序列**：
    1.  **[点击]** -> `我：妈妈，对不起打扰了，能不能借我50块钱？我快没东西吃了。`
    2.  **[点击]** -> `( 状态栏：对方正在输入... 持续3秒 )`
    3.  **[点击]** -> `妈：我有自己的家庭。`
    4.  **[点击]** -> `( 提示音效：对方已开启朋友验证，你还不是他（她）朋友。 )`
    5.  **[点击]** -> **手机框像灰尘一样解构散开，线条垂落在屏幕底部。**

#### **阶段二：画室的呓语 (The Last Portrait)**
* **视觉**：左侧出现猫的 1px 轮廓。中心开始缓慢生长出五彩自画像。
* **背景图**：`/assets/scenes/portrait_full.png` 作为居中背景显示，配合 SVG 自画像叠加层。
* **交互序列**：
    1.  **[点击]** -> `女主：它甚至不愿意为了这50块钱维持那个虚假的圆圈。`
    2.  **[点击]** -> `女主：没关系，阿喵。我有你，我还有这幅画。`
    3.  **[点击]** -> `女主：这块红色是太阳的颜色，虽然我只在旧平板的广告位里见过它。`
    4.  **[点击]** -> `女主：最后……一笔。阿喵，你看，我把自己画得多么完整。`
    5.  **[自动触发]** -> **沉闷的枪响。全屏剧烈颤抖（Glitch）。自画像中心炸裂。全屏转黑，色彩瞬间抽离。**
        * **动效**：枪响瞬间对 `portrait_full.png` 应用 `filter: grayscale(100%)` 和强烈的位移抖动（Glitch），红色闪烁叠加层 + SVG噪点滤镜。

#### **阶段三：意识荒原 (The Void Pact)**
* **视觉**：黑暗中只有 1px 的白色噪点。自画像变为死灰色的轮廓，在背景中忽明忽暗。
* **背景图**：`/assets/scenes/death_moment.png` 渐显，呈现低角度模糊视感（`blur(2px) brightness(0.3) scaleY(0.8) translateY(20%)`）。猫的 1px 轮廓叠加于图片之上。
* **交互序列**：
    1.  **[点击]** -> `猫：你的画还没画完，对吗？`
    2.  **[点击]** -> `女主：...`
    3.  **[点击]** -> `猫：她们拿走了你的命，我把它还给你。但代价是，你要成为我。`
    4.  **[点击]** -> `猫：让我成为你的心，去拿回那些本该属于你的颜色。`
    5.  **[点击]** -> `女主：...`
    6.  **[点击]** -> **猫的轮廓解构，化为无数细线汇聚。心跳声（低频震动）。**

#### **阶段四：手术台：强制夺舍 (The Awakening)**
* **视觉**：画面极暗，模拟濒死视角的模糊感。
* **自动/点击触发 [三次闪烁机制]**：
    1.  **[闪烁 1]**：屏幕亮起 0.2s。模糊的 1px 手术灯影。
        * `医生声音：准备关机，这个载体已经冷了。`
    2.  **[闪烁 2]**：屏幕亮起 0.4s。近距离看到医生俯身的轮廓线。
        * `医生声音：等等，它的心率...在以非生物频率跳动？`
    3.  **[闪烁 3]**：**全屏强白光（1.5s）**。伴随高频电子噪音。
        * `[ 系统日志：正在建立神经连接... ]`
        * `[ 载体：外科医生 / 状态：被接管 ]`

#### **阶段五：视角翻转 (The Flip)**
* **视觉**：白光散去。利用 `rotateX` 或视觉平移，将场景从"仰视"瞬间转变为"俯视"。
* **交互序列**：
    1.  **[视觉]**：玩家（现在是医生）正低头看着手术台上那个胸口插着机械、死去的女孩。
    2.  **[点击]** -> `女主（通过医生的口音）：...这是，我？`
    3.  **[点击]** -> `( 颤抖 )`
    4.  **[自动触发]** -> **审计终端从底部升起。**
    5.  **[点击]** -> `[ 警告：审计系统已察觉异常语义。请输入首次职业自检指令以维持同步。 ]`

### 3. 标题页转场 (Title Transition)

* **触发条件**：序幕阶段五第 2 步完成后，`setScene('title')` 跳转至标题页。
* **视觉**：
    - 背景图：`/assets/scenes/title_screen.png`，`brightness(0.4) contrast(1.2)`，叠加扫描线与微弱噪点。
    - 中央：`[ 死而替生 ]`，font-mono，tracking-[0.3em]，white/90。
    - 底部：`点击以初始化载体连接...`，呼吸闪烁动画。
* **交互**：点击屏幕任何位置，平滑淡出（1.5s）后 `setScene('surgery')`。
* **组件**：`src/scenes/Title.jsx`

### 4. 第一幕：手术室身份审计系统

#### **游戏机制**
* **起始条件**：完成序幕，`identityLevel` 设为 1，自画像第一路径永久变灰。
* **生命值系统**：
  - 初始 3 分，代表身份同步度
  - 每次 AI 审计判定为"不符合身份"扣 1 分
  - 分数归零 → 触发全屏 Glitch 效果 → 本幕重启（重置当前场景变量）
  - 成功通过审计（累积足够正确选择）→ 进入下一场景（电梯）
* **交互方式**：
  - 自由文本输入框
  - 输入完成后提交给 AI 审计系统
  - 显示审计结果（分数变化和判定理由）
* **审计 AI 配置**：
  > "你现在是赛博都市的身份审计防火墙。当前身份：外科医生。检测以下语句是否符合该职业逻辑：{玩家输入}。返回 JSON { 'isValid': boolean, 'reason': '分析报告' }"

#### **技术实现要求**
* **状态管理**：`useGameStore` 需要管理 3 分制审计分数
* **场景切换**：序幕结束后切换到 `scene: 'title'`（标题页），点击标题页后切换到 `scene: 'surgery'`
* **失败处理**：分数归零时调用 `triggerSceneReset()` 触发 Glitch 动画
* **成功推进**：审计成功时调用 `incrementIdentityLevel()` 推进游戏进度
