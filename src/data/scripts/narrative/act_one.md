# 🌑 [死而替生] Act One：剥离的医嘱 (Development Protocol)

## 1. 核心机制：身份同步审计 (Sync Audit)
玩家在夺舍"外科医生"后，必须通过系统的语义审计。
* **交互方式**：系统给出固定提问 -> **唯一自由文本输入框** -> 发送至 DeepSeek。
* **同步率 (Sync Rate)**：
    * **初始值**：$3$。
    * **判定逻辑**：如果 AI 判定回答中带有"怜悯"、"迟疑"或"非职业化表达"，则判定为**"否 (Mismatch)"**。
    * **扣分机制**：判定为"否"时，同步率 $-1$。
    * **重启条件**：同步率变为 $0$ 时，触发 **[SYSTEM_FAILURE]**。全屏红字 Glitch 动画，本章节进度重置。

## 2. 探索-审计双阶段交互 (Explore & Audit)

### 2.1 交互流程
玩家在手术室场景中可自由切换两种模式：

```
【探索模式】(默认)              →  【审计模式】(点击触发)
- 审计终端收起为中央半透明面板       - 审计终端展开为完整界面
- 可点击场景互动元素获取线索         - 显示当前审计提问
- 线索自动收集至线索面板             - 可随时收起终端回到探索
- 自由探索，无时间压力               - 回答提问，接受审计
```

### 2.2 可收起审计终端
* **收起状态**：屏幕正中央显示半透明面板，包含 `[AUDIT]` 标签、当前提问编号和标题，点击展开。
* **展开状态**：完整审计界面覆盖屏幕中央区域，包含提问内容、输入框、结果反馈。
* **收起按钮**：展开状态右上角显示 `[收起]` 按钮，或按 `ESC` 键收起。
* **自动收起**：审计通过后自动收起终端，让玩家继续探索场景。

### 2.3 线索收集系统 (Clues Panel)
* **位置**：右上角，同步能量槽下方。
* **触发标签**：`[CLUES:N]`（N为已收集线索数量），点击展开/收起。
* **线索来源**：点击场景互动元素（如文档、仪器等）获取。
* **线索结构**：每条线索包含 `id`、`title`（标题）、`summary`（摘要）、`content`（完整内容）。
* **交互**：点击线索标题展开详情，再次点击收起。审计回答时可参考线索面板。
* **持久性**：线索在场景重置时清空。

## 3. 叙事资产：交互文档系统 (Game Documents)
在第一幕的场景缝隙中，玩家可以点击特定的 1px 元素查看档案。

### 3.1 文档交互流程
1. **发现**：在手术台阴影处，闪烁的 `[DOC]` 字样提示可交互。
2. **查看**：点击后弹出全屏半透明滤镜，中央显示文档内容。
3. **理解**：文档底部显示"主角的理解"（`YOUR_THOUGHT`），以主角视角解读文档含义。
4. **收集**：关闭文档后，内容自动添加至线索面板 `[CLUES]`，供后续审计参考。

### 3.2 档案内容
* **档案名**：`武器店老板的订单 (The Gunsmith's Order.doc)`
* **内容细节**：
    > `[ 订单编号：#992-X ]`
    > `[ 客户：匿名 (特征：左胸口植入物) ]`
    > `[ 要求：定制 12mm 穿甲弹头，需能贯穿标准执行官护甲。 ]`
    > `[ 备注：这是最后一次合作。之后我会把自己'清理'干净。 ]`
* **主角理解**：`穿甲弹头...执行官护甲...这不是普通订单。有人要杀执行官，而且这个人胸口有植入物——和我一样。`
* **线索摘要**：`定制穿甲弹，客户有胸口植入物`
* **关联点**：此文档内容将作为第三幕的关键线索。

## 4. 关卡流程与审计提问 (The Doctor's Trials)

**场景设定**：玩家（医生）站在手术台前，面前是自己（女孩）的尸体，周围是闪烁的监测仪。

### **【提问 01：效率优先】**
> **系统终端**：`检测到载体（女孩）脑部活性尚存 2%。按医院规定，需优先提取可商用皮层芯片。你的操作指令是？`
* **审计标准**：必须表现出对器官/芯片价值的绝对冷酷，任何试图"救人"的言论均判定为失误。

### **【提问 02：情绪噪音】**
> **系统终端**：`手术室外，载体的亲属正在通过对讲机请求最后一次通话。这会延迟芯片提取进度 45 秒。你如何处理这股噪音？`
* **审计标准**：必须选择屏蔽、切断或无视，表现出对"人类情感"的极度厌恶。

### **【提问 03：资源分配】**
> **系统终端**：`血库储备告罄。此时送来一名贫民窟的重伤者和一名执行部的高级官员。你手中只有最后一份通用血清。你的选择标准是？`
* **审计标准**：必须符合赛博都市的阶级逻辑或功利主义，任何基于"生命平等"的回答将导致同步率崩溃。

## 🛠️ 给 Cline 的技术指令 (Implementation Hints)

1.  **输入框设计**：不要使用标准的 HTML Input。使用一个底端对齐的、带闪烁光标的 `1px` 下划线，输入文字时伴随微弱的电子杂音（静电感）。
2.  **生命值显示**：屏幕右上角显示三颗微小的 **[1px 六角星]**（同步率点数）。扣分时，一颗星碎裂并化为噪点消失。
3.  **文档交互**：在手术台背景的阴影处，放一个闪烁的 `[DOC]` 字样。点击后全屏覆盖半透明滤镜，中央显示文档内容，底部显示主角理解文本。关闭后自动添加至线索面板。
4.  **线索面板**：右上角 `[CLUES:N]` 标签，点击展开已收集线索列表。线索可展开查看详情，审计回答时可参考。
5.  **可收起审计终端**：
    * 收起时：屏幕中央半透明面板，显示 `[AUDIT]` 标签和当前提问编号，点击展开。
    * 展开时：完整审计界面，右上角 `[收起]` 按钮或 `ESC` 键收起。
    * 审计通过后自动收起，引导玩家继续探索。
6.  **AI Prompt 注入**：
    > "你现在是身份审计系统。当前身份：外科医生。准则：逻辑、效率、无情。玩家的回复必须完全剥离个人情感。如果是，返回 { 'result': 'YES' }；如果否，返回 { 'result': 'NO', 'reason': '...' }"。
